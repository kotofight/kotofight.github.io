# git版本控制工具总结

## 上班第一天

### 准备
```javascript
git config --global user.name "用户名"
git config --global user.email "邮箱"
```

### 第一步
装git 拉代码
```javascript
git clone 地址
```


### 第二步
修改文件后纳入版本控制，放入暂存区
```javascript
git add 文件名 ||git add .
```

### 第三步
在本地生成管理记录，放入本地版本库
```javascript
git commit -m "备注"
```

### 第四步
提交到服务器，把代码推送到远程
```javascript
git push origin 分支名
git push origin master
```

### 补充命令
查看日志
```javascript
git log//看以前的日志
git reflog //看所有的
```

查看状态
```javascript
git status
```

## 集中式和分布式
集中式 只有服务器有完整的版本，每个人的电脑都只有最新的版本
分布式 每个人的电脑都有完整的版本

## 本地回滚操作
```javascript
git reset --hard 版本号(哈希值、sha值)
```

## 上班第二天：拉同事的代码
```javascript
git pull origin 分支名
```

## 项目主管要做的事情
```javascript
git branch//查看所有分支
git branch 分支名//创建分支
git checkout 分支名//切换分支

//切换到master
git merge dev//把开发分支合并

//打标签
git tag v主版本号.此版本号.修订号
```
